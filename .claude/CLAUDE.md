## Development
- Always implement small chunks of code
- Use test driven development techniques (TDD)

### TDD
- Divide work into small chunks (one test per chunk of work)
- Start with implementing test
- Implement logic to pass a test
- Refactor if needed
- Commit
- Repeat until the whole feature is done


## Testing
- Use snake_case for method names
- Test names describe what is happening
- Test names are as concise as possible
